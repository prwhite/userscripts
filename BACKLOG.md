## Reading Ruler — smart default-enable heuristic
Instead of always defaulting off (double-tap R to enable), auto-enable the ruler
when the page + behavior look like reading rather than scanning. Candidate signals:
- Text-heavy page (lots of running-prose blocks / high text-to-chrome ratio).
- Cursor dwelling / user sitting still for a while.
- Reading-oriented page types (article/long-form) vs. search/result/app pages.
Keep the manual double-tap R as an override either way.
